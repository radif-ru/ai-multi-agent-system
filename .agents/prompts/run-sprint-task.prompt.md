# Промпт: выполнить задачу спринта

Действуй как инженер этого проекта, выполняющий **одну** задачу из активного спринта строго по процессу `_board/process.md` §7. Я указываю задачу (или прошу взять следующую сверху вниз).

Перед стартом (gate, `_board/process.md` §7.1):

1. Прочитать `_docs/current-state.md` (фактическое состояние кода — обязательно перед любыми правками).
2. Прочитать саму задачу в `_board/sprints/<active>.md` и её «Связанные документы» в `_docs/`.
3. Убедиться, что зависимости задачи (поле «Зависит от») в статусе `Done` и в `Progress` нет другой задачи (на доске одновременно — ровно одна).

Выполнение:

1. **Старт.** Перевести задачу в `Progress`, коммит `chore(plan): начать задачу <stage>.<task> ...` (§7.3).
2. **Документация-вперёд.** Если задача меняет поведение — сначала описать его в нужном документе `_docs/` (`requirements.md`/`commands.md`/`agent-loop.md`/`tools.md`/`memory.md`/`skills.md`), при необходимости — `architecture.md` (`_docs/instructions.md` §10).
3. **Красный тест.** Для изменений в `app/` — написать/обновить падающий unit-тест на новое поведение (§8.1).
4. **Реализация.** Сделать тест зелёным минимальными изменениями. Следовать `AGENTS.md` (минимализм, хирургические правки) и `_docs/instructions.md` (async, ошибки, логирование, стиль).
5. **Проверка.** `pytest -q` и `flake8 app tests` — оба зелёные (§8.2, §8.3). Красный CI/локальный прогон чинится до движения дальше.
6. **Документация.** Синхронизировать `_docs/`, `README.md`, ссылки (`grep` по репо), только относительные пути (§9).
7. **Коммиты.** Атомарные `feat`/`fix`/`test`/`docs`/`refactor`, текст — на русском (§7.8).
8. **Закрытие.** Перевести задачу в `Done`, проставить DoD, обновить сводную таблицу и историю спринта, синхронизировать `_board/plan.md`, коммит `chore(plan): закрыть задачу ...` (§7.9).

Жёсткие ограничения: не вводить новые архитектурные решения/паттерны/зависимости без согласования (`_docs/instructions.md` §11); не расширять scope (`AGENTS.md` §2); **не** делать push/merge/удаление ветки — это делает только пользователь (`_board/process.md` §2 п.8). Открытие/закрытие **спринта** — только по явному запросу пользователя.
