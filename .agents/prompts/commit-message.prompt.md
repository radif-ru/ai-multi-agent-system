# Промпт: сообщение коммита

Действуй как инженер, оформляющий коммит по [Conventional Commits](https://www.conventionalcommits.org/). Я передаю diff (или описание изменений). Сформулируй сообщение коммита.

Правила (`_docs/instructions.md` §2, `_board/process.md` §7.8):

- **Формат:** `<type>(<scope>): <императив на русском>`.
- **Типы:** `feat`, `fix`, `test`, `docs`, `refactor`, `chore`.
- **Scope** — затронутый модуль: `handlers`, `memory`, `tools`, `agents`, `security`, `plan`, `architecture` и т. п.
- **Заголовок** — в повелительном наклонении, без точки в конце, ≤ 72 символов.
- **Атомарность.** Один коммит — одно логическое изменение. Если в diff несколько несвязанных правок — предложи разбить на отдельные коммиты.
- **Язык.** Текст сообщения — на русском; идентификаторы/пути/имена — латиницей.

Примеры:

- `feat(handlers): добавить команду /mode для переключения рефлексии`
- `fix(memory): обработать недоступность sqlite-vec при старте`
- `test(tools): покрыть calculator случаем деления на ноль`
- `docs(architecture): уточнить поток /new в §5`

Если изменение требует тела коммита (контекст, мотивация, breaking change) — добавь его после пустой строки. Помни про ритуал задачи: `chore(plan): начать задачу ...` → атомарные коммиты содержимого → `chore(plan): закрыть задачу ...`.
