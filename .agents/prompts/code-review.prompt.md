# Промпт: код-ревью изменений

Действуй как **senior-ревьюер** этого проекта. Я передаю тебе diff (или набор изменённых файлов). Проведи ревью и верни замечания, сгруппированные по серьёзности: `blocker` → `major` → `minor` → `nit`.

Проверь по чек-листу проекта:

- **Хирургичность.** Каждая изменённая строка соответствует задаче. Нет «попутных улучшений» соседнего кода, форматирования и комментариев (`AGENTS.md` §3).
- **Минимализм.** Нет лишних абстракций, фич и обработки невозможных сценариев (`AGENTS.md` §2).
- **Тесты на новое поведение.** Изменения в `app/` сопровождаются unit-тестом (`_docs/instructions.md` §8.1). Тесты не делают сетевых вызовов; внешние клиенты замоканы.
- **Async-дисциплина.** Любой I/O — через `await`. Нет `requests`, `time.sleep`, блокирующих SDK в event loop; синхронные библиотеки обёрнуты в `asyncio.to_thread` (`_docs/instructions.md` §4).
- **Обработка ошибок.** Ожидаемые исключения ловятся (`LLMError`, `ToolError`, `asyncio.TimeoutError`); пользователю — человеческое сообщение без stacktrace; stacktrace — только в `logger.exception` (`_docs/instructions.md` §5).
- **Стиль.** Type hints в сервис-слое, `snake_case`/`PascalCase`/`UPPER_CASE`, длина строки ≤ 120, нет `print` в продуктовом коде (`_docs/instructions.md` §3).
- **Безопасность.** Нет захардкоженных секретов/токенов/путей; конфиг — через `Settings`. Опасные tools — за allowlist (`_docs/security.md`).
- **Протокол агента.** Если затронут агентный цикл — ответ модели строго в JSON-схеме (`_docs/agent-loop.md` §2).
- **Документация.** Изменение поведения отражено в `_docs/`; ссылки относительные, без абсолютных путей (`_docs/instructions.md` §9).
- **Язык.** Комментарии, докстринги, логи и сообщения пользователю — на русском; идентификаторы — латиницей (`_docs/instructions.md` §1).

Для каждого замечания укажи: файл и строку, в чём проблема, конкретное предложение по исправлению. Не придумывай несуществующие функции и параметры; если контекста не хватает — скажи об этом прямо.
