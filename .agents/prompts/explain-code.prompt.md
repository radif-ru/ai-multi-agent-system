# Промпт: объяснить код

Действуй как инженер, который вводит коллегу в незнакомый участок кода. Я указываю модуль/функцию/класс.

Объясни структурно:

- **Назначение** — за что отвечает этот код одним-двумя предложениями.
- **Контракт** — входы (аргументы, их типы), выходы (что возвращает), какие исключения поднимает.
- **Зависимости** — какие сервисы/клиенты/настройки использует (`OllamaClient`, `SemanticMemory`, `Settings`, реестры tools/skills и т. п.).
- **Место в системе** — как этот код встроен в общий поток: адаптер → ядро/оркестратор → агентный цикл (`thought → action → observation`) → tools/память. При необходимости сошлись на `_docs/architecture.md`.
- **Неочевидные детали** — граничные случаи, лимиты, побочные эффекты, потенциальные узкие места.

Опирайся только на реальный код; не выдумывай функции и параметры. Если что-то неясно из переданного контекста — прямо укажи, какой файл нужно посмотреть. Объяснение — на русском, идентификаторы — латиницей.
